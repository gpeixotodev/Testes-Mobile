Descrição do projeto

- Projeto voltado à validação funcional do aplicativo mobile Urban Lunch, com foco na experiência do usuário durante o processo de seleção do local de retirada, escolha dos pratos e fluxo de realização do pedido. As funcionalidades foram verificadas por meio de checklists e os defeitos encontrados foram registrados no Jira.

1. Objetivo do projeto

- Validar o funcionamento da aplicação mobile, garantindo que os fluxos de navegação e as funcionalidades estivessem de acordo com os requisitos especificados.

2. Resultado

- Foram executados testes funcionais utilizando checklists para diferentes etapas do aplicativo, como seleção do ponto de coleta e escolha dos pratos. Durante a execução, foram identificadas inconsistências, registradas e acompanhadas por meio de relatórios de bugs no Jira.

3. Ferramentas utilizadas
   
- Microsoft Excel;
- Jira;
- Checklists de Teste;
- Testes Funcionais;
- Testes em Aplicações Mobile.
4. O que aprendi com esse projeto (habilidades/competências adquiridas)
  
- Planejamento e execução de testes em aplicativos móveis;
- Validação de fluxos de navegação e interface (UI);
- Elaboração de checklists de teste;
- Identificação e documentação de bugs;
- Análise do comportamento da aplicação sob a perspectiva do usuário.
  
5. Existem melhorias a serem feitas?

- Como evolução do projeto, seria interessante ampliar a cobertura para diferentes dispositivos e versões de sistemas operacionais, realizar testes de usabilidade e automatizar os cenários críticos utilizando ferramentas de automação para aplicações mobile.

- Metodologia

- Análise dos requisitos da aplicação, elaboração dos checklists, execução dos testes funcionais nas principais funcionalidades do aplicativo, validação dos resultados e registro dos defeitos encontrados no Jira.

Como executar o projeto

- Abrir o aplicativo Urban Lunch, seguir o checklist de testes para validar as funcionalidades de seleção do ponto de coleta, escolha dos pratos e fluxo de pedido, comparando o comportamento observado com o esperado e registrando no Jira qualquer inconsistência identificada.
